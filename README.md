## Introduction to Wordle

Developed by Josh Wardle, Wordle (released in 2021) is a single player word game where players have six attempts to guess a five-letter word ([Founder](https://theygotacquired.com/gaming/wordle-acquired-by-the-new-york-times/)). 2,000,000 users play Wordle daily and it currently serves a cultural role in daily-life for Americans ([Players](https://www.seisen.com/student-life/seisen-post/features/~board/seisen-post/post/why-is-wordle-not-popular-anymore)). Wordleâ€™s logic and frontend is designed using Javascript, HTML, and CSS ([Syntax](https://syntax.fm/show/430/creator-of-wordle-josh-wardle)).

To play the original Wordle, users are given a 6x5 grid where they can only access one row at a time. Each row gives users a space to guess the word that is generated by the system each day. To use a turn, each user must guess a letter for each box in the row. If the userâ€™s guessed letter is in the right position as the systemâ€™s letter, the box in the row turns green. If the userâ€™s guessed letter is not in the right position but is in another position of the systemâ€™s word, the box in the row turns yellow. If the userâ€™s guessed letter is simply not in the systemâ€™s word, then the box in the row turns grey. If the user guesses all letters of the systemâ€™s word correctly, all boxes turn green and the user is redirected to a congratulatory web page. If the user does not guess the word, the guessed word is revealed to the user on the top of the 6x5 grid.

## Choosing Word
We created an array of 8 possible strings. At the start of a game, an index is chosen randomly. We used the rand8 function, and shifted it to only use the 3 most significant bits. Thus, the random integer will always be less than 8. To ensure complete randomization, we continuously generate random integers until the user enters the game from the start screen. Once the user has entered the game, the random integer is used to index into the array of possible words, and choose a specific word as the answer to that game.


## Cycling through letters
We used vram_put to start at letter â€˜Aâ€™ and move through the alphabet by simply adding an integer to this memory location. We use an array of 5 integers to keep track of the offset from â€˜Aâ€™ of each letter so that when the user moves back and forth, they can edit the letters from where they previously left off. Additionally, we inhibit the user from cycling before â€˜Aâ€™ or after â€˜Zâ€™, by simply limiting the offset to be between 0 and 25, inclusive. 

```
void cycle_letters(i,x,y) {
  ppu_off();
  vram_adr(NTADR_A(x, y));
  vram_put('A' + i);
  ppu_on_all();
}
```

## Moving to next letters
The user presses the right and left buttons to move back and forth between letters. If these buttons are pressed, the x-position of the cursor is simply incremented 5, if moving to the right, or decremented 5, if moving to the left. This allows the user to move between letters to edit them before submitting their guess. 

## Guessing and Checking word
We used the space key or A on the game controller to allow the user to submit their guess for checking. When the game detects this pad trigger, it begins word checking. The entered word is saved as an array of characters, and each letter in the userâ€™s guess is checked against every letter in the true answer. Using a loop, we check every letter the user has entered and fill the box the correct color. Additionally, we move the cursor down to the start of the next word, by simply changing the x and y-position, to allow the user to make their next guess. 

## How the game ends 
We used two local variables, game_over and tries, in our game loop to keep track of the status of the game. At the start of each guess, this game_over is initialized to 0. While the game checks the correctness of each letter, we increment this variable each time the game finds a completely correct letter. Thus, if the user correctly guesses the word, this variable will be 5. Furthermore, after each guess, the local variable, tries, is incremented by 1. This helps us keep track of how many guesses the user has made. The main game loop and the game ending are separated into a larger if/else statement. If the user exhausts all their tries or if one guess is correct, the game enters the ending condition and prints the game over screen. 

## Conclusion
Overall, we were successfully able to recreate the main aspects of Wordle on the NES. Our main limitation was due to the color palette. Our game logic became quite complicated due to the number of variables we needed to keep track of. Therefore, we decided to only use one palette in the entirety of our game to keep this aspect relatively easy to manage. In doing this, we were unable to mimic the original game by coloring incorrect letters grey. Rather, we do not change the color to display to the user the inaccuracy of the letter.
